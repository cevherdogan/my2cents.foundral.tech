Foundral Investment Atlas v4.1 — Operational Guide

Mobile-first runbook: how to configure, run, validate, automate, and release with tags.

1) Configure

config.yaml
cp config/config.example.yaml config/config.yaml
# Key fields
data.use_live_fetch: true|false
scoring.macro_regime_hint: CUTS|HIKES|STICKY|UNKNOWN
scoring.weights: trend/survival/macro/events
thresholds: drawdown_breach / pick_flip_margin
notifications: enabled + slack_webhook_url

2) Run locally

macOS
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
bash scripts/run_daily.sh
open site/report.html

3) Validate outputs

sanity checks
site/report.html
data/derived/latest_signals.json
data/logs/runs.jsonl
site/history/report_*.html

4) Release: commit + tag + push

Git
git add .
git commit -m "v4.1: integration layer prototype (TTD vs ENPH)"

git tag -a v4.1 -m "Foundral Investment Atlas v4.1"
git push -u origin main
git push origin --tags

If you haven't set a remote yet:

git remote add origin <YOUR_GIT_URL>
git branch -M main

5) Automate daily

GitHub Actions
# Put this file in:
.github/workflows/atlas.yml

# Start from:
ops/github_actions_workflow.yml

For secrets: keep config out of git; inject via GitHub Secrets in the workflow (v4.2+).
